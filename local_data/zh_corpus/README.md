# Local Chinese Corpus Seed

This directory is an isolated local data workspace for building a Chinese visual document retrieval benchmark.

It is intentionally kept outside the upstream `colpali/` and `vidore-benchmark/` trees.

## Raw Datasets

The initial selected datasets are copied from:

```text
/mnt/d/data/cert_vie/04.VIE_data/02_KIE_data_ok
```

Current local copies:

| Dataset | Local path | Images | JSON files | Role |
|---|---:|---:|---:|---|
| XFUND-zh-val | `raw/XFUND-zh-val` | 50 | 100 | Highest-quality Chinese full-page form seed |
| COLD_CELL_600 | `raw/COLD_CELL_600` | 600 | 1200 | Larger Chinese cell/table/document seed |
| EPHOIE_SCUT_311 | `raw/EPHOIE_SCUT_311` | 311 | 622 | Chinese extraction seed, likely cropped/strip-like pages |

Total: 961 images and 1922 JSON files.

## Boundary

Do not modify files under `raw/`. Treat them as copied source data.

Future generated artifacts should go under:

```text
processed/
benchmark/
benchmark_mineru/
reports/
```

## MinerU OCR/Layout Enrichment

The official MinerU Python SDK is installed in `.venv-vidore` as `mineru-open-sdk`.

Flash mode uses MinerU's token-free official API and writes outputs under `mineru/flash/`:

```bash
source env.sh
python scripts/enrich_zh_corpus_with_mineru.py --scope selected --mode flash --limit 5 --overwrite --no-proxy
```

Notes:

- `--scope selected` enriches only pages that currently have selected benchmark queries.
- `--scope all` enriches the full 961-page seed corpus.
- `--limit 0` disables the smoke-test limit.
- `--no-proxy` is needed on this machine because the current local proxy breaks TLS to `mineru.net`; direct access works.
- Precise `extract` mode requires `MINERU_API_TOKEN` or `MINERU_TOKEN`.

Current MinerU flash outputs:

| Dataset | Markdown pages |
|---|---:|
| XFUND-zh-val | 49 |
| COLD_CELL_600 | 561 |
| EPHOIE_SCUT_311 | 1 |
| Total | 611 |

Raw API JSON files are in `mineru/flash/raw/`, and Markdown files are in `mineru/flash/markdown/`.

## Evaluation

Use the local evaluation wrapper for this MinerU benchmark:

```bash
source env.sh
python scripts/evaluate_zh_mineru_benchmark.py --validate-only --limit 1
```

```bash
source env.sh
python scripts/evaluate_zh_mineru_benchmark.py \
  --model-name vidore/colpali \
  --batch-query 1 \
  --batch-doc 1 \
  --batch-score 1
```

Smoke runs:

```bash
bash run_zh_mineru_colpali_smoke.sh
bash run_zh_mineru_siglip_smoke.sh
```

The evaluator loads `benchmark_mineru/vidore_compat_test.jsonl`, casts the local `image` paths to PIL images, drops rows with `query: null` by default, and writes metrics under `outputs/zh_corpus_mineru/`.

For a larger negative pool that includes pages without selected queries, pass `--include-unqueried`.

The SigLIP baseline may require `protobuf` in the virtual environment.

## BM25 and Hybrid Evaluation

BM25 and ColPali+BM25 fusion are implemented in an isolated local script:

```bash
source env.sh
python scripts/evaluate_zh_mineru_hybrid.py --mode bm25
python scripts/evaluate_zh_mineru_hybrid.py --mode rrf --model-name vidore/colpali
python scripts/evaluate_zh_mineru_hybrid.py --mode linear --model-name vidore/colpali --alpha 0.5
```

Convenience wrappers:

```bash
bash run_zh_mineru_bm25.sh
bash run_zh_mineru_colpali_bm25_rrf.sh
bash run_zh_mineru_colpali_bm25_linear.sh
```

The BM25 implementation is self-contained and uses only `query` plus MinerU `text_description`; it does not require an embedding model or reranker. Hybrid modes keep the official ColPali implementation untouched and combine scores only in the local evaluation wrapper.

Current BM25-only full result is written to:

```text
outputs/zh_corpus_mineru/bm25_zh_corpus_mineru_bm25_metrics.json
```
